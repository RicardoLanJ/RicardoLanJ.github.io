CImg的直方图均衡化

发表于 2017-03-14

CImg的多通道存储时， 方式是RRRGGGBBB， 可参考下文
http://www.cs.ukzn.ac.za/~hughm/ap/cimg/html/reference/group__cimg__storage.html

均衡化的关键语句为

cimg_rof(*this,ptrd,T) {
    const int pos = (int)((*ptrd-vmin)*(nb_levels - 1.)/(vmax-vmin));
    if (pos>=0 && pos<(int)nb_levels) *ptrd = (T)(vmin + (vmax-vmin)*hist[pos]/cumul);
  }

cimg_rof是宏，对data数组遍历。data是public的。

因为cimg存储图像为RRRRGGGBBBB的格式， 所以直接对整个data数组做均衡化即可。彩色图像的均衡化有几种方法，1）分别对rgb通道进行均衡化，可能会出现奇怪的情况。2）使用强度分量均衡化，如先转换为his模型。由于cimg的存储方式，所以对整个data均衡化相当于 255 R+G+B）/（Cumulative 3）= 255 * H / Cumulative。
其中H = (r+g+b)/3。

[uwp]GoBack() parameter

发表于 2017-03-11

Situation

uwp 左上角的后退按钮实现通常为

SystemNavigationManager.GetForCurrentView().AppViewBackButtonVisibility =
            ((Frame)sender).CanGoBack ?
                AppViewBackButtonVisibility.Visible :
                AppViewBackButtonVisibility.Collapsed;

和

Frame rootFrame = Window.Current.Content as Frame;
       if (rootFrame.CanGoBack)
       {
           e.Handled = true;
           rootFrame.GoBack();
       }

UWP的frame在navigate时是new一个（用cache好像可以保留）， 调用onNavigateTo， page间传递数据依靠navigate传递的参数，问题在goback传了什么。

problem

ex在mainpage修改后跳转到newpage直接点左上角返回， viewmodel（例）中的数据都丢失了， 因为这是new的typeof（mainpage）, 一个新页面。

这个问题在回退到自己导航去的页面时没有出现。

?

goback是怎么传参数的， 为什么接下来可以自动传viewmodel

debug

断点设置为onnavigate， 发现回到mainpage时e.parameter=””， 和onlaunch中一样。newpage中parameter类型为viewmodel。得出结果goback的参数类型等于navigate来时的参数。所以大概是goback会寻找一个同类型的符合的参数自动传， 大概吧
（= 。 =）

solution

所以在APP的onlaunch中， 把navigate到mainpage的参数修改为你传递的数据类型。

    if (e.PrelaunchActivated == false)
{
    if (rootFrame.Content == null)
    {
        // 当导航堆栈尚未还原时，导航到第一页，
        // 并通过将所需信息作为导航参数传入来配置
        // 参数
        rootFrame.Navigate(typeof(MainPage), new ViewModels.TodoItemViewModel());
    }
    // 确保当前窗口处于活动状态
    Window.Current.Activate();
}

haskell的几个练习

发表于 2016-10-24

其实是总结…

####1
QuickCheck要update cabel才能装

下面的句子可以执行所有quickcheck
{-# LANGUAGE TemplateHaskell #-}
return [] – need this for GHC 7.8
– quickCheckAll generates test cases for all ‘prop_*’ properties
main = $(quickCheckAll)

qucikcheck里 ==> 可以用于限制条件

infix infixl infixr用于设置运算符优先级

####2
haskell里没有循环，都是利用尾递归或者列表生成式，所以在进行迭代时如果次数太多内存会爆炸。尝试过各种方法，都会这样。找了很久发现一个seq函数， 函数原型是a -> b -> b，它的作用是立即计算a然后返回b。这就可以实现循环执行同一个操作，如下的牛顿迭代，可以观察发现内存不会无限增长

squareroot :: Float -> Float -> Integer -> Float
squareroot _ x0 0 = x0
squareroot r x0 n =  x `seq` squareroot r x (n-1)
    where x = dd r x0

####3
haskell 的\$和.运算符区别在于， \$用于去掉括号，在连续的使用函数时，f (g x) = f $ g x，而.更主要的目的是函数的链式调用， 可以把几个函数组合成一个函数，ex：

func = map (+ 1) . reverse
func [1,2,3]

map (+ 1) . reverse $ [1,2,3]

map (+ 1) $ reverse  [1,2,3]

他们的效果都一样
