[uwp]GoBack() parameter

Situation

uwp 左上角的后退按钮实现通常为

SystemNavigationManager.GetForCurrentView().AppViewBackButtonVisibility =
            ((Frame)sender).CanGoBack ?
                AppViewBackButtonVisibility.Visible :
                AppViewBackButtonVisibility.Collapsed;

和

Frame rootFrame = Window.Current.Content as Frame;
       if (rootFrame.CanGoBack)
       {
           e.Handled = true;
           rootFrame.GoBack();
       }

UWP的frame在navigate时是new一个（用cache好像可以保留）， 调用onNavigateTo， page间传递数据依靠navigate传递的参数，问题在goback传了什么。

problem

ex在mainpage修改后跳转到newpage直接点左上角返回， viewmodel（例）中的数据都丢失了， 因为这是new的typeof（mainpage）, 一个新页面。

这个问题在回退到自己导航去的页面时没有出现。

?

goback是怎么传参数的， 为什么接下来可以自动传viewmodel

debug

断点设置为onnavigate， 发现回到mainpage时e.parameter=””， 和onlaunch中一样。newpage中parameter类型为viewmodel。得出结果goback的参数类型等于navigate来时的参数。所以大概是goback会寻找一个同类型的符合的参数自动传， 大概吧
（= 。 =）

solution

所以在APP的onlaunch中， 把navigate到mainpage的参数修改为你传递的数据类型。

    if (e.PrelaunchActivated == false)
{
    if (rootFrame.Content == null)
    {
        // 当导航堆栈尚未还原时，导航到第一页，
        // 并通过将所需信息作为导航参数传入来配置
        // 参数
        rootFrame.Navigate(typeof(MainPage), new ViewModels.TodoItemViewModel());
    }
    // 确保当前窗口处于活动状态
    Window.Current.Activate();
}

本来是e.parameter，然而他为“”
